================================================================================
                          CÓMO USAR EN EXPOSICIÓN
================================================================================

MINUTO 0 (5 minutos antes):
──────────────────────────
Verifica que todo funciona:

    python scripts/demo_startup.py

Esperado: Todos los checks en VERDE ✓

MINUTO 1 (Iniciar contenedores - 30 segundos):
──────────────────────────────────────────────

Opción A (Manual - Recomendado):
    docker-compose -f docker-compose.pgvector.yml up -d
    docker-compose -f docker-compose.milvus.yml up -d
    # Espera 15-20 segundos

Opción B (Automático):
    .\scripts\start_demo.ps1 -Containers

MINUTO 2 (Iniciar API - 20 segundos):
────────────────────────────────────

Abre una NUEVA terminal (Ctrl+Shift+`) y ejecuta:
    uvicorn app.main:app --reload

Verifica en: http://127.0.0.1:8000/docs (debe estar disponible)

MINUTO 3 (DEMO EN VIVO - Elige UNA opción):
───────────────────────────────────────────

OPCIÓN A - DEMO INTERACTIVA (Recomendado):
    En terminal nueva:
    python scripts/demo_comparison.py
    
    Menú:
    1. Crear documentos
    2. Listar documentos
    3. Ejercicio: Búsqueda en 'AI'
    4. Ejercicio: Búsqueda en 'Databases'
    5. Búsqueda personalizada
    6. Actualizar documento
    7. Eliminar documento

OPCIÓN B - SWAGGER UI (Más profesional):
    Abre: http://127.0.0.1:8000/docs
    
    Pasos sugeridos:
    1. POST /pgvector/documentos (Crear)
    2. GET /pgvector/documentos (Listar)
    3. POST /pgvector/search (Buscar)
    4. Repetir lo mismo en /milvus
    5. Compara tiempos de respuesta

OPCIÓN C - ANÁLISIS DE RENDIMIENTO:
    En terminal nueva:
    python scripts/performance_test.py
    
    Menú:
    1. Crear documentos en bulk (50)
    2. Pruebas de búsqueda (10 iteraciones)
    3. Búsquedas con filtro
    5. Ejecutar análisis complete

================================================================================
                              TROUBLESHOOTING
================================================================================

Si PostgreSQL no conecta:
    docker-compose -f docker-compose.pgvector.yml restart

Si Milvus no conecta:
    docker-compose -f docker-compose.milvus.yml restart
    # Espera 20+ segundos, es lento al inicializar

Si API no funciona:
    # En terminal: Ctrl+C para terminar
    uvicorn app.main:app --reload

Si necesitas limpiar todo:
    docker-compose -f docker-compose.pgvector.yml down -v
    docker-compose -f docker-compose.milvus.yml down -v
